# 📚 Audiobook Bot

Телеграм-бот для быстрого доступа к **аудиокнигам** и их **PDF-версиям**.
Книга может быть плоским списком глав или разбита на **части/разделы**
(как «Лухуф» — 3 части, 61 глава); экран выбора части показывается только
если частей больше одной.

Бот: [@international_universityBook_bot](https://t.me/international_universityBook_bot)

---

## Что уже сделано

- 📖 **Каталог книг** с пагинацией, карточка книги, список из 15 глав.
- 🎧 **Прослушивание глав** — аудио отправляется по `file_id` (мгновенно, без
  повторной загрузки), с навигацией ⬅️/➡️ и отметками ✅ прослушанных глав.

  конца (нативное авто-продолжение для аудио-сообщений).
- 📄 **PDF целиком** одной кнопкой.
- ▶️ **«Продолжить с главы N»** — бот запоминает прогресс.
- 🌐 **Два языка интерфейса** (RU/EN) с переключателем — под международный вуз;
  добавить язык = добавить один файл словаря.
- 🔍 **Поиск** по названию/автору + **inline-режим** (`@бот запрос` в любом чате).
- ✍️ **Подпись при пересылке** — см. раздел ниже.
- 🎨 **Новый формат стилей** — см. раздел ниже.
- 🛠 **Админ-панель заложена в архитектуру**, но намеренно не реализована
  (см. `ARCHITECTURE.md` и `app/handlers/admin.py`).

## Быстрый старт (локально, Windows / Linux)

```bash
cd D:\projects\university-audiobook-bot       # на Linux — соответствующий путь
python -m venv .venv
.venv\Scripts\python -m pip install -r requirements.txt   # Linux: .venv/bin/python

copy .env.example .env        # Linux: cp .env.example .env
#  .env уже содержит ваш BOT_TOKEN; впишите ADMIN_IDS, при желании BRAND_NAME

.venv\Scripts\python -m scripts.seed     # наполнить демо-каталогом (2 книги)
.venv\Scripts\python -m app              # запустить бота
```

Откройте бота в Telegram, нажмите **/start**.

> Для деплоя на VPS — см. **`deploy/DEPLOY.md`** (Docker или systemd).

## ✍️ Подпись при пересылке («с какого бота переслано»)

**Да, это возможно — и реализовано.** Но с важной оговоркой про то, как
устроен Telegram:

- Бот **не может** вмешаться в сам момент пересылки. Когда пользователь
  пересылает сообщение, Telegram копирует его **дословно** и сверху добавляет
  свою служебную строку «Переслано от \<Бот\>» (её текст менять нельзя, а
  отправитель может её вообще скрыть).
- Поэтому надёжный способ гарантировать подпись **именно снизу и нашим текстом**
  — это **встроить подпись в само сообщение/подпись к файлу**. Подпись к аудио
  и к PDF «путешествует» вместе с файлом при пересылке, так что получатель
  всегда видит, откуда это.

Что делает бот: к каждой главе (аудио), PDF и карточке книги снизу добавляется
футер:

```
——————————————
📚 Аудиотека
🔗 Слушать в боте: @international_universityBook_bot
```

Ссылка — это **deep-link** (`?start=book_<slug>`): получатель по ней попадает
сразу на нужную книгу. То есть пересылка работает как канал привлечения новых
пользователей.

Код — в `app/formatting/rich.py` (`signature`, `with_signature`,
`caption_with_signature`). Подпись локализована (RU/EN) и обрезается так, чтобы
не превысить лимит подписи Telegram (1024 символа). Текст бренда меняется
переменной `BRAND_NAME` в `.env`.

## 🎨 Новый формат стилей

По вашим ссылкам речь о двух вещах:

1. **Rich-сущности** (Bot API 7.4+): раскрывающиеся цитаты
   `<blockquote expandable>`, спойлеры, кастомные эмодзи, жирный/курсив и т.д.
   — **полностью поддерживаются** и используются в интерфейсе (приветствие,
   карточка книги и др.).
2. **Rich Messages / `sendRichMessage`** (Bot API **10.1**, вышло
   **11.06.2026**) — совсем новое. Обёртки в библиотеках (aiogram, PTB) ещё
   дозревают, а полная схема параметров метода официально пока не опубликована.

**Инженерное решение:** интерфейс построен на надёжных rich-сущностях (HTML),
а путь `sendRichMessage` **заложен и включается флагом** `USE_RICH_MESSAGES`
в `.env` (по умолчанию `false`) с **автоматическим откатом** на HTML при любой
ошибке. Так бот не падает из-за двухнедельного API, но готов перейти на него,
как только схема устаканится. См. `app/formatting/rich.py` → `send_text` и
класс `SendRichMessage`.

## 📥 Добавление материалов (аудиокниги + PDF)

Бинарные файлы **не** хранятся в репозитории/БД — хранится только Telegram
`file_id`. Файл загружается в Telegram **один раз**, дальше раздаётся мгновенно.

1. В `.env` задайте `MEDIA_CHAT_ID` — чат, куда бот может писать (ваш личный
   чат после /start, либо приватный канал, где бот — админ).
2. Разложите файлы:

   ```
   media/<slug>/cover.jpg          (обложка, необязательно)
   media/<slug>/book.pdf           (полный PDF)
   media/<slug>/01.mp3 ... 15.mp3  (главы; имя файла = номер главы)
   media/<slug>/meta.json          (метаданные; нужны, чтобы создать новую книгу)
   ```

   Пример `meta.json`:
   ```json
   {
     "title": {"ru": "Введение в экономику", "en": "Introduction to Economics"},
     "author": "J. Doe",
     "description": {"ru": "...", "en": "..."},
     "sort_order": 1,
     "chapter_titles": {"1": {"ru": "Глава 1", "en": "Chapter 1"}}
   }
   ```
3. Запустите импорт:
   ```bash
   .venv\Scripts\python -m scripts.import_media
   ```
   Скрипт загрузит файлы, заберёт их `file_id` и пропишет в БД. Повторный запуск
   безопасен.

### Первая книга — «Лухуф» (3 части, 61 глава)

Материалы лежат в папке `ЛУХУФ. СКОРБИ КЕРБЕЛЫ/` (файлы вида `1.3. Название.mp3`,
части кодируются первым числом). Отдельный скрипт сам разбирает имена на
часть/номер/название:

```bash
.venv\Scripts\python -m scripts.import_luhuf      # без MEDIA_CHAT_ID — сухой прогон (показывает план)
# затем, задав MEDIA_CHAT_ID в .env:
.venv\Scripts\python -m scripts.import_luhuf      # загрузка обложки + PDF + 61 главы
```

До появления реальных файлов главы без аудио показываются как 🔒 «скоро».

## ⚙️ Конфигурация (`.env`)

| Переменная | Назначение |
|---|---|
| `BOT_TOKEN` | Токен от @BotFather |
| `ADMIN_IDS` | ID админов через запятую (для будущей админ-панели) |
| `BRAND_NAME` | Бренд в подписи-футере |
| `DEFAULT_LANGUAGE` | `ru` или `en` — язык по умолчанию |
| `DATABASE_URL` | По умолчанию SQLite; легко заменить на Postgres |
| `USE_RICH_MESSAGES` | Включить экспериментальный `sendRichMessage` |
| `PROTECT_CONTENT` | Запретить пересылку/сохранение (нам не нужно → `false`) |
| `MEDIA_CHAT_ID` | Чат для импорта `file_id` (только для `import_media`) |

## ⚠️ Важное

- **Включите inline-режим**, чтобы заработал поиск-через-`@бот` в любых чатах:
  @BotFather → `/setinline` → выбрать бота → задать подсказку. Сейчас он
  выключен (`can_inline=False`).
- **Токен был передан в открытом виде** при постановке задачи. Перед продакшеном
  желательно отозвать его в @BotFather (`/revoke`) и вписать новый в `.env`.
- Запускайте **один** экземпляр бота на токен (иначе конфликт `getUpdates`).
